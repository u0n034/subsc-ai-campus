# 第2章 セクション2：【実践1】AIによる高速市場調査（3C分析）

## **(0:15:00 - 0:55:00) セクション2：【実践1】AIによる高速市場調査（3C分析）**

### **2.1 コンセプト：3C分析の「新しい常識」 (0:15:00 - 0:20:00)**

**(0:15:00)**

⭐️さあ、いよいよ実践に入ります。
AIディレクターとしての皆さんの、最初のミッションは「3C分析」です。

3Cとは、「Customer（市場・顧客）」「Competitor（競合）」「Company（自社）」の頭文字です。
これは、マーケティング戦略の設計において、最も基本的かつ本質的なフレームワークの一つです。

なぜ、3C分析は「基本」なのでしょうか。
それは、この3つの視点こそが、ビジネスの成否を決定づける「すべて」だからです。

**Customer（市場・顧客）**を理解しなければ、誰に届ければいいのか、そして何を求められているのかが分かりません。
**Competitor（競合）**を分析しなければ、私たちの「独自性」や「差別化ポイント」、すなわち「なぜ競合ではなく、私たちを選ぶべきなのか」という問いに答えられません。
**Company（自社）**の強みを定義しなければ、私たちが顧客に「約束できる価値」が何なのか、明確にできません。

この3つの要素を明らかにすることで、初めて私たちは「誰に」「何を」「どう伝えるか」という、LPの根幹を設計できるのです。
⭐️**3C分析なくして、戦略なし。戦略なくして、成果なし。**
これは、AI時代になろうとも変わらない、ビジネスの絶対法則です。

⭐️そして、セクション1で学んだ通り、AI時代の3C分析は「AIで高速にたたき台を生成」→「人間が批判的に検証」→「AIに再指示」という、超高速のイテレーションサイクルへと変わりました。
AIは「速度」を、人間は「判断」を提供します。

**(0:19:00)**

⭐️では、具体的に、どうやってAIと「協業」して3C分析を実行するのか。
そのプロセスを、今から、実際に体験していただきます。

本講座では、皆さん全員が同じプロジェクトを題材に、実践演習を行います。
そのプロジェクトとは、こちらです。

**【架空プロジェクト】**
『地方の中小企業（社員10〜30名）をターゲットにした、月額5万円のAI活用コンサルティングサービス』のLP制作

このプロジェクトは架空ですが、現在の日本において、極めて現実的で、需要の高いビジネスです。
中小企業のAI導入支援という、タイムリーで複雑な課題を題材にすることで、皆さんは「本物」のマーケティング思考を身につけることができます。

さあ、準備はよろしいでしょうか。
AIという最強のリサーチャーに、私たちの最初の指示を出しましょう。

### **2.2 ワークショップ（実践）：AIに「3C分析」を実行させる (0:20:00 - 0:55:00)**

**(0:20:00)**

#### **【ステップ0：役割定義（Role Assignment）】AIディレクションの「黄金ルール」**

⭐️実践に入る前に、極めて重要な「AIディレクションの黄金ルール」をお伝えします。

**AIの出力品質は、プロンプトの質で決まる。**
これは第1章で学んだ、AIディレクションの鉄則です。

そして、高品質なプロンプトの第一要素は、「タスク（何をしてほしいか）」ではありません。
それは「役割定義（Role Assignment）」です。

AIに優れた仕事をさせたいなら、まずAIに「あなたは誰なのか」「どのような専門性を持った存在として振る舞うべきなのか」を、明確に定義する必要があります。

⭐️なぜでしょうか。
AIは「役割」を与えられることで、その役割に応じた「知識」「思考様式」「出力の文体やトーン」を適応させるようにプログラムされているからです。

例えば、「あなたは経験豊富なマーケティングコンサルタントです」と定義すれば、AIは「コンサルタント」らしい、専門的で戦略的な視点で分析を行おうとします。
一方、「あなたは初心者向けの教育者です」と定義すれば、AIは分かりやすく、噛み砕いた説明をしようとします。

この「役割定義」は、毎回、AIとの新しいセッション（会話）を開始する際に、**必ず最初に行うべき、最も重要な「儀式」**です。

では、実際のプロンプトを見てみましょう。

**(0:22:30)**

**【プロンプト例（役割定義）】**

```
あなたは、BtoBマーケティング、特に中小企業向けSaaSおよびコンサルティングサービスの市場動向に精通した、経験豊富なマーケティングコンサルタントです。

これから、私（AIディレクター）が企画する新しいサービス（地方の中小企業向けAI活用コンサルティング）のランディングページ制作を手伝ってください。

あなたのタスクは、私の指示に基づき、専門的な視点から市場分析、競合分析、ペルソナ設計、そして訴求メッセージの考案を行うことです。

まずは、3C分析からお願いします。
```

このプロンプトの構造を分解してみましょう。

**1行目：専門性の定義**
「あなたは、BtoBマーケティング、特に中小企業向けSaaS…に精通した、経験豊富なマーケティングコンサルタントです。」

ここで、私たちはAIに「BtoBマーケティング」「中小企業向けSaaS」「コンサルティングサービス」という、今回のプロジェクトに直結する専門領域を明示しました。
これにより、AIは、その領域に関連する知識を優先的に引き出そうとします。

**2行目：プロジェクトの文脈（コンテキスト）**
「これから、私が企画する新しいサービス（地方の中小企業向けAI活用コンサルティング）のランディングページ制作を手伝ってください。」

ここで、AIに「何のために」この分析を行うのかという「目的（ゴール）」を与えました。
AIは、単に一般論を語るのではなく、「LP制作」という明確なゴールに向けて、出力を最適化しようとします。

**3行目：タスクの全体像**
「あなたのタスクは、私の指示に基づき、専門的な視点から市場分析、競合分析、ペルソナ設計、そして訴求メッセージの考案を行うことです。」

ここで重要なのは、「私の指示に基づき」という一文です。
これにより、AIは「一方的に分析して終わり」ではなく、私たちAIディレクターからの「指示」や「フィードバック」を受け入れる「協業モード」に切り替わります。

**4行目：最初の指示**
「まずは、3C分析からお願いします。」

ここで、AIに最初のタスクを指示しました。

この「役割定義プロンプト」をAIに入力した瞬間、私たちとAIの間に「契約関係」が成立します。
AIは、今この瞬間から、「BtoBマーケティングに精通した、経験豊富なコンサルタント」として、私たちに協力してくれる準備が整いました。

**(0:25:00)**

#### **【ステップ1：市場・顧客（Customer）分析】顧客の「悩み」と「期待」を言語化する**

役割定義が完了しました。
次に、3C分析の最初のC、「Customer（市場・顧客）」の分析を、AIに実行させます。

ここで、私たちがAIから引き出したい情報は何でしょうか。
それは、顧客の「表面的な属性（年齢や性別）」ではありません。

私たちが本当に知りたいのは、顧客の「心の中」です。
彼らが抱えている「悩み」とは何か。
彼らがAIに「期待している成果」とは何か。
そして、彼らが私たちのサービスを導入する際に感じる「障壁（不安や懸念）」とは何か。

この3つを明らかにすることで、私たちは「顧客の言葉」で、顧客に「刺さる」メッセージを設計できるようになります。

では、実際のプロンプトを見てみましょう。

**【プロンプト例（Customer分析）】**

```
タスク: 3C分析の「市場・顧客 (Customer)」の分析を実行してください。

制約条件:
1. ターゲット: 地方在住、従業員10～30名規模の企業の経営者およびIT担当者。
2. 業界: 製造業、小売業、建設業（非IT系の中小企業）。
3. 分析項目:
   * 彼らが現在抱えている「生成AI」に関する主な悩みや課題（例：何から手をつければいいか分からない、導入コストが不安など）。
   * 彼らがAIに期待している具体的な成果（例：業務効率化、コスト削減、人手不足の解消など）。
   * 彼らがAIコンサルティングサービスを導入する際の主な障壁（例：費用対効果が見えない、専門的すぎて理解できないなど）。
4. 出力形式: 重要なポイントを箇条書きで整理してください。
```

**(0:27:30)**

このプロンプトの構造を分解しましょう。

**「タスク」の明確化**
「3C分析の『市場・顧客 (Customer)』の分析を実行してください。」
何をすべきかを、一文で明確に定義しました。

**「制約条件」による精度のコントロール**
ここが、このプロンプトの核心部分です。
「制約条件」とは、AIの出力を私たちの意図に沿った方向に「誘導」するための「ガードレール」です。

1番目の制約「ターゲット」では、「地方在住」「従業員10〜30名」「経営者およびIT担当者」と、顧客の属性を極めて具体的に定義しました。
これにより、AIは「一般的な日本企業」ではなく、「地方の小規模企業」という、より解像度の高い分析を行おうとします。

2番目の制約「業界」では、「製造業、小売業、建設業（非IT系）」と指定しました。
なぜ「非IT系」とわざわざ付け加えたのでしょうか。
それは、IT系企業とそれ以外の企業では、AIに対する「リテラシー」や「導入障壁」が全く異なるからです。
この一言を加えることで、AIの分析精度は劇的に向上します。

3番目の制約「分析項目」では、私たちがAIに分析してほしい「質問」を、3つに分解して明示しました。
「悩み」「期待」「障壁」という3つの視点です。

そして4番目、「出力形式」では、「箇条書きで整理してください」と指定しました。
これは小さな指定に見えますが、極めて重要です。
なぜなら、AIは指示がなければ、長文の散文で出力してくることがあり、それでは私たちが後で「使いづらい」からです。
箇条書きで出力させることで、私たちはその情報を、後の工程（ペルソナ設計や訴求メッセージ作成）で「再利用」しやすくなります。

**(0:33:00)**

いかがでしょうか。

AIは、わずか数秒で、地方の中小企業が抱えるAI導入に関する「悩み」「期待」「障壁」を、驚くほど具体的に、そして網羅的に分析してくれました。

ここで極めて重要なのは、AIディレクターとしての皆さんが、この出力を見た時に取るべき「態度」です。

**「すごい！AIは完璧な分析をしてくれた！」と感動して終わってはいけません。**

思い出してください。
第1章で学んだ、AIディレクターのコアスキル2、「批判的検証（Critical Validation）」です。

AIディレクターは、AIの出力を決して鵜呑みにしません。
私たちは今、この出力を「批判的」な視点で検証します。

**検証ポイント1：「ハルシネーション（嘘）」は含まれていないか？**
例えば、「既存システムとの連携障壁（オンプレや古いベンダー製品とのAPI連携困難）」という指摘。
これは本当でしょうか？
地方の中小企業の多くが、本当にオンプレミス（自社サーバー）で古いシステムを運用しているのでしょうか？

もし皆さんが、実際にこの市場に詳しいのであれば、あるいはクライアント企業にヒアリングできるのであれば、この点を「ファクトチェック」すべきです。
もしかしたら、実際にはクラウドサービスに移行している企業も多く、AIの分析は「やや古い認識」に基づいている可能性があります。

**検証ポイント2：「重要度」は正しく評価されているか？**
AIは、10個もの「悩み」をリストアップしてくれました。
しかし、これら10個すべてが、同じ「重さ」を持っているわけではありません。

AIディレクターの仕事は、このリストの中から、「最も深刻で、私たちのLPで真っ先に訴求すべき悩み」を「優先順位付け」することです。

例えば、「何から始めればよいか不明」という悩みと、「補助金・助成金の活用難易度」という悩み。
どちらが、より根本的で、より多くのターゲット顧客が抱えている悩みでしょうか？

恐らく、前者です。
なぜなら、補助金の話は「すでにAI導入を決めて、資金調達を検討している段階」の企業の悩みであり、私たちがターゲットにすべき「AI導入に二の足を踏んでいる企業」の最優先課題ではない可能性が高いからです。

このように、AIの出力を「材料」として受け取り、その中から「何を使うか」「何を捨てるか」「何をもっと深掘りすべきか」を判断するのが、AIディレクターの「品質管理力」です。

**(0:37:00)**

ここで、もう一つ重要な「AIディレクションテクニック」をお伝えします。
それは「追加質問」、あるいは「深掘り指示」です。

もし、AIの最初の出力に、私たちが納得できない点や、もっと具体的に知りたい点があれば、私たちは躊躇なく、AIに「追加の指示」を出すべきです。

例えば、こんな追加プロンプトが考えられます。

```
一点、追加で分析してほしいのですが、「何から始めればよいか不明」という悩みについて、もう少し具体的に教えてください。
例えば、彼らは具体的にどのような「最初の一歩」で躓いているのでしょうか？
業務の洗い出しでしょうか？ツールの選定でしょうか？それとも社内の合意形成でしょうか？
```

このように、AIとの対話は「一問一答」で終わりではなく、「繰り返し（イテレーション）」によって、分析の「解像度」を上げていくプロセスなのです。

**(0:39:00)**

#### **【ステップ2：競合（Competitor）分析】競合の「強み」と「弱み」を可視化する**

Customer分析が完了しました。
次に、3C分析の二つ目のC、「Competitor（競合）」の分析に進みます。

なぜ、競合分析が重要なのでしょうか。
それは、私たちのサービスが市場で選ばれるためには、「競合との違い」、すなわち「差別化ポイント」を明確にする必要があるからです。

顧客は常に「比較」します。
「この会社のサービスと、あの会社のサービス、どっちが自分に合っているのか？」と。

もし私たちが、競合の「強み」と「弱み」を理解していなければ、顧客のこの問いに答えることはできません。
競合分析とは、「なぜ、顧客は競合ではなく、私たちを選ぶべきなのか？」という問いへの答えを見つける作業です。

では、実際のプロンプトを見てみましょう。

**【プロンプト例（Competitor分析）】**

```
タスク: 3C分析の「競合 (Competitor)」の分析を実行してください。

前提情報:
• 私たちのサービスは「月額5万円」という、中小企業でも導入しやすい価格帯の「伴走型コンサルティング」です。

分析項目:
1. 競合のセグメント: この市場における競合を、以下の3つに分類してください。
   a. 大手コンサルティングファーム（高価格帯）
```
